Laravel Linux Commands

1. Installing Laravel
composer global require laravel/installer

Installs the Laravel installer globally on your system.

2. Creating a New Laravel Project
laravel new project_name

Creates a new Laravel project named project_name.

3. Serve the Application
php artisan serve

Starts a local development server at http://localhost:8000.

4. Running Migrations
php artisan migrate

Executes the migrations defined in your project.

5. Creating a Controller
php artisan make:controller ControllerName

Generates a new controller named ControllerName.

6. Database Seeding
php artisan db:seed

Seeds the database with records specified in your seed classes.

7. Creating a Migration
php artisan make:migration create_table_name

Creates a new migration file for creating a database table.

8. Rolling Back Migrations
php artisan migrate:rollback

Rolls back the last database migration.

9. Clearing Cache
php artisan cache:clear

Clears the application cache.

10. Optimizing Class Loading
composer dump-autoload -o

Optimizes the autoloader for better performance.

11. Creating Routes
Route::get('/example', 'ExampleController@index');

Defines a route that maps a URL to a controller action.

12. Artisan Commands List
php artisan list

Displays all available Artisan commands.

13. Generating Model Factories
php artisan make:factory ModelFactoryName

Creates a new model factory.

14. Managing Environment Variables
cp .env.example .env

Copies the example environment file to create a new environment file.

15. Running Tests
php artisan test

Executes PHPUnit tests for your Laravel application.

Laravel Tricks and Tutorials

1. Setting Up Laravel

Follow these steps to set up Laravel:

  1. Install Composer from here.
  2. Run composer global require laravel/installer.
  3. Create a new project with laravel new project_name.
  4. Navigate to the project directory and start the server with php artisan serve.
2. Using Eloquent ORM

Eloquent ORM provides an easy way to interact with your database. Here's how you can use it:

  • Create a model with php artisan make:model ModelName.
  • Define your database schema in migration files.
  • Use the model to interact with the database in your controllers.
3. Creating Middleware

Middleware allows you to filter HTTP requests. Here's how to create one:

  • Run php artisan make:middleware MiddlewareName to create a new middleware.
  • Add logic to the handle method of the middleware.
  • Register the middleware in app/Http/Kernel.php.
